MCP Airtable Server

Model Context Protocol server enabling AI assistants to seamlessly interact with Airtable databases through a standardized interface.

View on GitHub Documentation

Features

01

Full CRUD Operations

Create, read, update, and delete records with complete Airtable API coverage.

02

Schema Inspection

Explore base and table structures programmatically (Enterprise plan).

03

Advanced Filtering

Use Airtable formulas for complex queries and data retrieval.

04

Batch Operations

Efficient bulk record operations for high-performance workflows.

05

OAuth 2.0 Support

Modern authentication with automatic token refresh capabilities.

06

Attachment Upload

Direct file uploads to Airtable or external storage (S3/GCS).

Quick Start

// Claude Desktop Configuration
{
  "mcpServers": {
    "airtable": {
      "command": "npx",
      "args": ["mcp-airtable"],
      "env": {
        "AIRTABLE_API_KEY": "your_api_key_here"
      }
    }
  }
}

Authentication Methods

Priority 1

Tool Parameter

Pass the API key directly in tool arguments.

{ "airtableApiKey": "patXXX..." }
Priority 2

HTTP Headers

Use headers for remote MCP server deployments.

x-airtable-api-key: patXXX...
Authorization: Bearer patXXX...
Priority 3

Environment Variable

Set in your environment for local STDIO transport.

AIRTABLE_API_KEY=patXXX...
